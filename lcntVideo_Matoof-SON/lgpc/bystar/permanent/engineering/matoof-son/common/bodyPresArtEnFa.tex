\pnote{
  
Greetings.

This is Mohsen Banan.

I am a software and telecom and internet engineer.

The title of this presentation is:
``soncli.mcs: An After-Market Open-Source Enabler''
``An Evolutionary Bridge From Nokia EdenNet SON to More Than SON''

Over the past two decades or so, we have developed the PyCS (Python Command Services)
Framework. A component of the PyCS Framework is MaTOOF (Modules and Targets Oriented Open
Fabric). MaTOOF's abstractions and model are very similar to Nokia's EdenNet SON
abstractions and model. It turns out that it is actually quite easy to ``absorb'' EdenNet
SON into the PyCS Framework. The strategy mapping EdenNet SON onto MaToof revolves around
``soncli.mcs''. Soncli.mcs is an external execution environment for ``playing'' SON
modules. Think on SON modules as CDs and think of a EdenNet SON Web UI as a CD-Player.
In that context Our soncli.mcs is another CD-Player which is capable of ``Playing'' all of
your exisiting SON modules.
EdenNet SON includes a python framework that they call the ``EdenNet SON SDK''
which is really more than just a Software Development Kit. In fact the SDK is External
Execution Facility (EEF). soncli.mcs maps the SDK model onto CLI. We can then build on it on our
own. Nokia need not be involved. All of this can be done as an aftermarket capability.
The benefits are enourmous.

}

\begin{small}
\begin{columns}[T]
\column{0.5\textwidth}

\begin{center}
        \colorbox{blue!30}{\color{black} \textbf{Strength:}}
\end{center}

\begin{itemize}
  \item Choice of Python (12 years ago)
  \item Modules Architecture for RAN Systems Management
  \item Targets Architecture for Cells
  \item Python API for RAN and Telecom  Sys Mgmt Abstractions
\end{itemize}

\column{0.5\textwidth}

   \begin{center}
      \colorbox{blue!60}{\color{white} \textbf{Weaknesses:}}
    \end{center}

    \begin{itemize}
      \item Jailed in the Telecom Walled Garden
      \item Abstraction of Targets is Limited to Cells
      \item Limited in Scope to RAN and Telecom
      \item Limited to a a single Modules Player (Web-UI)
      \item No CLI for Modules Execution

    \end{itemize}

\end{columns}
\end{small}

\begin{exampleblock}{With soncli.mcs We Can Liberate Modules and Targets}
  \begin{itemize}
    \item Inside of the Telecom Walled Garden Open Up and Extend EdenNet SON
    \item Profit from the Nokia SON Experience of ``Modules and Target Oriented Open Fabric''
          (MaTOOF) for Systems Management Outside of the  Telecom Walled Garden
  \end{itemize}
\end{exampleblock}
