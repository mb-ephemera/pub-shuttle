    \end{block}

\pnote{
  soncli.mcs is a CLI tool, but it is also a foundation for something larger.

  Nokia EdenNet SON distinguishes between Native Modules --- those designed to run
  inside the EdenNet Web UI platform --- and External Modules --- those that use
  the SDK but execute outside the platform entirely.

  Because soncli.mcs exposes SON capabilities through the command line and through
  the full BISOS-PyCS framework, it becomes possible to build complete standalone
  External SON Applications on top of it.

  These applications can have their own graphical user interfaces, their own data
  pipelines, their own reporting systems --- entirely separate from the Nokia platform.
  Nokia does not need to be involved in their development or deployment.

  The progression is: Native Modules running in the Nokia Web UI, then External
  Modules running through soncli.mcs, and finally full External SON Applications
  that stand entirely on their own.

  This is the full arc of what the aftermarket strategy enables.
}
